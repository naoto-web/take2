# Take2 を GitHub Pages で公開する手順

このフォルダの中身をそのままアップロードすれば、スマホに「ドン!」アイコンでインストールできるPWAになる。
gitのインストールは不要。すべてブラウザで完結する。

## ファイル構成（8ファイル）

| ファイル | 役割 |
|---|---|
| index.html | ゲーム本体（全部入り1ファイル） |
| manifest.webmanifest | PWA設定（アプリ名・アイコン・全画面表示） |
| sw.js | オフライン対応（Service Worker） |
| icon-192.png / icon-512.png | 通常アイコン（案5・ドン!） |
| icon-maskable-192.png / icon-maskable-512.png | Android丸マスク用（中身を84%に縮小した版） |
| apple-touch-icon.png | iPhone用アイコン（180px） |

## 手順

### 1. GitHubアカウント（持っていなければ）
1. https://github.com/signup を開く
2. メールアドレス・パスワード・ユーザー名を登録（無料プランでOK）
3. ユーザー名がそのまま公開URLになる（例：`naoto-tax` → `naoto-tax.github.io`）

### 2. リポジトリ作成
1. GitHubにログイン → 右上の「+」→「New repository」
2. Repository name：`take2`
3. **Public** を選択（PrivateだとPagesが無料プランで使えない）
4. 「Create repository」

### 3. ファイルをアップロード
1. 作成直後の画面にある「**uploading an existing file**」リンクをクリック
2. このフォルダの8ファイルを全部ドラッグ&ドロップ
3. 下の「Commit changes」ボタンを押す

### 4. GitHub Pages を有効化
1. リポジトリ上部の「Settings」タブ →左メニューの「Pages」
2. 「Branch」を `None` → `main` に変更（フォルダは `/ (root)` のまま）→「Save」
3. 1〜2分待つと上部に公開URLが表示される：
   `https://ユーザー名.github.io/take2/`

### 5. スマホにインストール
1. スマホの**Chrome**で上のURLを開く（LINE内ブラウザではなく）
2. 「⋮」→「**アプリをインストール**」（または「ホーム画面に追加」→「インストール」）
3. 今度はChromeバッジなし・「ドン!」アイコンで追加される
4. 前に作ったバッジ付きショートカットは長押し→削除でOK

## 更新するとき

ゲーム（css/js_*.htmlのパーツ）を修正したら、Take2フォルダで `ビルド.ps1` を実行すると
このフォルダの index.html が作り直される（アカギの絵文字差し替えも自動）。
あとは新しい `index.html` をリポジトリにアップロードし直すだけ。
（リポジトリのファイル一覧で index.html を開く → 鉛筆アイコンの隣の「…」→「Delete」してから再アップロード、
または「Add file」→「Upload files」で同名アップロードすると上書きされる）

インストール済みのアプリは、次にオンラインで開いたときに自動で最新版になる。

## 注意

- 通算スコア（localStorage）はサイト単位の保存なので、GAS版から引き継がれず0からスタートになる
- URLを知っている人しかアクセスしないが、リポジトリ自体は世界中から見える「公開」状態になる
- 公開版はアカギの顔が絵文字😎に差し替え済み（元画像入りの身内版はGAS/ローカル確認用のみ）
